Styling Structure — Proposal

branch foundry-module · 2026-05-30 · splitting styles/dauligor-importer.css (7,228 lines) + a component finder

You want the one giant stylesheet split into per-area files plus a doc to find components fast. Here's the lay of the land and two ways to do it — I need your pick before I reorganize a 7k-line file I can't visually test myself.

The complication

The file isn't authored in clean per-component order — it interleaves:

  • spell-manager rules appear in 3 separate regions (~1942, ~3770, ~5432); feature-manager in 2; class-* scattered from line 564 to 5122.
  • There are ~10–15 cross-block bridge selectors — e.g. .dauligor-importer-app--sequence-prompt .dauligor-class-options__region, .dauligor-asi__toolbar .dauligor-class-browser__subtitle, .dauligor-feature-manager__spells-host > .dauligor-spell-manager (the embedded prep manager).

So a tidy per-component split means reordering rules, not just cutting the file at line marks. That's the crux of the choice below.

Two approaches

Approach A — safe ordered slice

Cut the file into ~10 contiguous chunks in their existing order and load them in that order. Because ordered concatenation === the original file, the cascade is byte-identical — zero visual risk. Downside: files stay partly mixed (you'd get spell-core.css, spell-main.css, spell-extra.css rather than one spell-manager.css), so the finder doc does the heavy lifting for "where's component X".

Approach B — clean per-component reorg recommended

Gather each window/app's rules into its own file (one spell-manager.css, one feature-manager.css, …). This is the real "ease of use" win. It requires reordering, so there's a small regression risk on the ~10–15 bridge selectors — mitigated by: load a tokens.css + base.css first; keep each window's full block contiguous; place each bridge selector in the containing window's file; and you reload Foundry to eyeball it after (the junction points here now, so that's quick). BEM naming means cross-block collisions are rare, so risk is low but non-zero.
