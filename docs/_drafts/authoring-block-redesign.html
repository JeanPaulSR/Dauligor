Authoring-side Block — redesign mockups

What a content-creator sees while building a block (the counterpart to the admin review). Static mockups — react / pick a direction, then I build it, reusing the review components from the admin side.

Today the authoring side is thin and scattered. You get a slim header strip (block name + a queue count + Save Progress), the block's actual contents only show as "(in this block)" markers inside each separate editor, and there's no single place to see, manage, or review your whole block before submitting.
  • No whole-block view — you build pieces across editors and submit blind.
  • No managing — can't see/remove a queued change from one place; Save vs Submit is unclear.
  • Weak picker / switching — the block chooser is a forced dialog with no context, and switching the active block is clumsy.
Three surfaces below address each. Sample throughout: a creator mid-build on a Druid class pack (class + Wild Shape + CR column + Animal Forms group → Wolf + Circle of the Land subclass).

Surface 1 — Block picker / switcher

Replaces the forced, context-free dialog. Your open blocks as cards (name · #changes · types · last edited) + create new. The active block also gets a switcher in the bar (Surface 2).

Choose a block to work in

Everything you author goes into the chosen block and is reviewed + approved together.

Surface 2 — Block bar + "Your block" workspace (the main change)

While authoring in any editor, a persistent bar shows the active block + a right-side drawer always lists what's in it — so you see your whole block as you build, remove pieces, and Save / Review & Submit from one place. Replaces the thin strip.

The bar's "switch block ▾" is the Surface-1 switcher inline. Build cost: medium — the drawer reuses the block's drafts the wrapper already tracks; remove = the existing draft-drop path.

Surface 3 — Review & Submit (your block, before sending)

"Review & Submit" opens the creator's own Option-C review of the whole block — the same split-pane the admin uses, so you see exactly what they'll see. Remove anything, then submit the lot for review.

My lean: all three, since they're one flow — Surface 2 (bar + drawer) is the everyday win (always see/manage your block while authoring), Surface 3 is the confident send-off (reuses the admin review), and Surface 1 is the clean entry/switch. Build order: 2 → 3 → 1.
