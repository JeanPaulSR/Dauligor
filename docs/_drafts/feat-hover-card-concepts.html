You have learned a number of spells that you can cast as rituals. These spells are written in a ritual book, which you must have in hand while casting one of them.

The pin interaction

  1. Hover a reference → the ephemeral card appears.
  2. Click the pin (top-right) → the card "pops out": it stops being tied to the hover, gains a drag bar + close button, and a Go to row.
  3. Drag the bar to move it anywhere on screen; open as many as you like.
  4. Close (× or unpin) any time.

This lets a DM park several references side-by-side while reading — e.g. pin Ritual Caster, a condition, and a spell while planning an encounter.

Shared behaviour

  • Fixed size — ~380px wide, body capped (~215px), scrolls inside; pinned windows keep the same size.
  • Name at the top in the accent colour (gold here; in-app it follows the active theme — blue on your dark theme).
  • Click the reference (hover state) navigates to the feat; in the pinned window, the Go to row does that job.
  • Dangling reference (target missing) → no card, just the unresolved badge.

Nested pop-ups

A card can spawn a child card without closing itself — so you can drill in without losing your place.

Full prerequisite
Dexterity 13 or higher, proficiency with light armor, and the Sneak Attack feature.

First use: the prerequisite line. If the combined requirements are too long for the card, the line truncates; hovering it opens a small child pop-up with the full text — and the parent card stays open. This generalises: a reference inside a card's description can open its own child card the same way. Rule: opening a child never closes its parent — cards stack, and dismissing a child returns you to the parent.
